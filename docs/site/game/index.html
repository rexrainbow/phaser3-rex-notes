    Resize¶

    game.destroy(width, height);

    Also fires game.events resize event.

    Destroy¶

    game.destroy();

    Also fires game.events destroy event.

    Main loop¶

    1. game.events prestep event
      1. trigger game.input.update()
      2. trigger game.sound.update()
    2. game.events step event
    3. SceneManager.update, for each active scene
      1. scene.sys.events preupdate event
        1. TweenManager.preUpdate() to arrange active targets
        2. UpdateList.preUpdate(), to arrange game objects in UpdateList
      2. scene.sys.events update event
        1. TweenManager.update(), to run active tweens
        2. UpdateList.update → gameObject.preUpdate
      3. scene.update
      4. scene.sys.events postupdate event
    4. game.events poststep event
    5. game.renderer.preRender()
    6. game.events prerender event
    7. SceneManager.render()
    8. game.renderer.postRender()
    9. game.events postrender event

    Note

    Each scene is a standalone system.

    Pause / Resume events¶

    • Pause (window is invisible) : game.events pause event
    • Resume (window is visible) : game.events resume event

    Global data¶

    Insatance of data manager in game.registry, or scene.registry in each scene.
